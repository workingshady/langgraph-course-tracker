## Repository layout

- `sections/` – one folder per course section (prefixed `01_` … `12_`).
  Inside each section:
  - `lectures/` – quick notes, summaries, links, screenshots.
  - `exercises/` – tiny coding drills, scratch pads, experiments.
  - `projects/` – section-specific mini builds.
- `projects/` – larger end-to-end builds or cross-section experiments.
- `assets/` – optional media (screens, diagrams, PDFs).
- `progress_tracker.md` – completion checklist for lectures/sections.
